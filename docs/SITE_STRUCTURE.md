# SITE_STRUCTURE.md — полная структура сайта «Амана»

Полная карта веб-приложения: все экраны, маршруты, навигация, содержимое каждого
экрана, действия, состояния и привязка к бэкенд-модулям. Фронтенд — SPA на React +
React Router; всё под `/app` требует авторизации (JWT), кроме зоны входа.

---

## 1. Тип и оболочка

- **SPA**, маршрутизация на React Router.
- **Зоны:**
  - **Публичная** — вход (без авторизации).
  - **Приложение** — общий каркас с боковым меню и верхней панелью, доступ по JWT.
- **App Shell (каркас авторизованной зоны):**
  - **Sidebar (левое меню):** Дашборд · Клиенты · Каталог · Договоры · Календарь ·
    Садака · Разработчикам · Настройки.
  - **Topbar (верхняя панель):** название организации · быстрый поиск · колокол
    напоминаний · меню пользователя (профиль, выход).
- **Сквозные состояния** для каждого экрана: загрузка (skeleton), пусто (empty state),
  ошибка, нет прав.

## 4. Детальное описание экранов

### 4.1 Вход — `/login`
- **Назначение:** аутентификация пользователя.
- **Элементы:** логотип «Амана», поля email + пароль, кнопка «Войти», ошибка входа.
- **Действия:** отправка → получение JWT → редирект на `/`.
- **Бэкенд:** `iam` (логин, выдача JWT; внутренний API `/api/app`).

### 4.2 Первичная настройка — `/setup`
- **Назначение:** первый запуск — создать организацию и владельца.
- **Элементы:** название организации, ФИО и доступ владельца, базовые настройки
  (валюта, координаты — Грозный по умолчанию).
- **Действия:** создать организацию → вход.
- **Бэкенд:** `iam`.

### 4.3 Дашборд — `/`
- **Назначение:** общая картина бизнеса.
- **Элементы (KPI-карточки):** активные договоры; суммарный остаток к получению;
  договоры с просрочкой; платежей ожидается на этой неделе; собрано садаки.
- **Блоки:** ближайшие напоминания (с пометкой «перенесено из-за намаза»);
  последние договоры; быстрые действия («Новый договор», «Новый клиент»).
- **Состояния:** пустой дашборд для новой организации с подсказками.
- **Бэкенд:** агрегирует `financing`, `scheduling`.

### 4.4 Клиенты — `/clients`
- **Назначение:** список и поиск клиентов.
- **Элементы:** строка поиска, таблица (ФИО, контакты, число договоров, остаток),
  кнопка «Новый клиент».
- **Действия:** открыть карточку, перейти к созданию.
- **Бэкенд:** `crm`.

### 4.5 Новый клиент — `/clients/new`
- **Элементы:** форма (ФИО, телефон, идентификационный документ для договора).
- **Действия:** сохранить → карточка клиента.
- **Валидация:** обязательные поля на клиенте (zod) и инварианты на бэкенде.
- **Бэкенд:** `crm`.

### 4.6 Карточка клиента — `/clients/:id`
- **Элементы:** данные клиента; вкладка «Договоры» (его договоры со статусами);
  вкладка «Платежи» (история); кнопки «Редактировать», «Оформить договор».
- **Бэкенд:** `crm`, `financing`.

### 4.7 Редактирование клиента — `/clients/:id/edit`
- **Элементы:** та же форма с предзаполнением. **Бэкенд:** `crm`.

### 4.8 Каталог — `/catalog`
- **Назначение:** товары, доступные для рассрочки.
- **Элементы:** таблица (название, категория, закупочная цена, **бейдж халяль-статуса**:
  халяль/харам/сомнительно), поиск, кнопка «Новый товар».
- **Бэкенд:** `catalog`.

### 4.9 Новый товар — `/catalog/new`
- **Элементы:** название, категория, закупочная цена, **обязательный халяль-статус**.
- **Логика:** статус «харам» допустим в каталоге, но блокирует оформление договора.
- **Бэкенд:** `catalog`.

### 4.10 Карточка / редактирование товара — `/catalog/:id`, `/catalog/:id/edit`
- **Элементы:** данные товара, бейдж статуса, действия. **Бэкенд:** `catalog`.

### 4.11 Договоры — `/contracts` ← ядро
- **Назначение:** все договоры рассрочки.
- **Элементы:** таблица (клиент, товар, цена продажи, остаток, статус, прогресс-бар,
  бейдж просрочки), фильтры по статусу, поиск, кнопка «Новый договор».
- **Бэкенд:** `financing`.

### 4.12 Мастер создания договора — `/contracts/new` ← ключевой UX
Пошаговый wizard:
- **Шаг 1 — Клиент:** выбрать существующего или создать на лету.
- **Шаг 2 — Товар:** выбрать из каталога; закупочная цена подтягивается; товар со
  статусом «харам» недоступен (с пояснением).
- **Шаг 3 — Условия:** наценка (суммой или %), первоначальный взнос, число платежей,
  периодичность, дата старта.
- **Шаг 4 — Предпросмотр:** автоматически построенный **график мурабахи** (таблица
  платежей с датами и суммами; расчёт на бэкенде через `POST /api/app/contracts/preview`,
  **без сохранения**) + блок **«Без рибы vs обычный кредит»** (recharts: фиксированная
  сумма против растущего долга) + итог «цена зафиксирована».
- **Шаг 5 — Подтверждение:** создать → договор в статусе «Активен» (create+activate в один
  шаг; статус «Черновик» отдельным экраном не показывается), переход в карточку.
- **Бэкенд:** `financing` (расчёт по skill `murabaha-engine`), `crm`, `catalog`.

### 4.13 Карточка договора — `/contracts/:id` ← главный экран продукта
- **Шапка:** клиент, товар, цена продажи, остаток, статус, прогресс.
- **Вкладка «График»:** список плановых платежей (срок, сумма, статус:
  оплачен / **частично оплачен** / предстоит / просрочен). Статусы долей производны от
  остатка (`Outstanding` — источник правды).
- **Вкладка «Платежи»:** история фактических платежей + кнопка **«Принять платёж»**
  (модальное окно с суммой; принимается **любая сумма ≤ остатка**; проверка «не больше
  остатка»).
- **Действие «Досрочно погасить»:** подтверждение → остаток в ноль, статус «Завершён»,
  **без штрафа**.
- **Действие «Отменить договор» (только владелец):** подтверждение → статус «Отменён».
- **Блок «Сравнение с кредитом»:** визуализация анти-риба.
- **Баннер просрочки (если есть):** кнопка **«Начислить садаку» (только владелец)**
  (фиксированная сумма, уходит в реестр благотворительности, **не меняет долг**). Менеджер
  баннер видит, но кнопка начисления ему недоступна.
- **Бэкенд:** `financing`.

### 4.14 Календарь — `/schedule`
- **Назначение:** планирование с учётом намаза.
- **Элементы:** календарь (день/неделя); **окна намаза подсвечены как заблокированные**;
  пятница показывает окно джума; задачи отображаются с пометкой переноса.
- **Действия:** создать задачу из слота.
- **Бэкенд:** `scheduling` (времена намаза через порт `PrayerTimesProvider`).

### 4.15 Напоминания — `/schedule/reminders`
- **Элементы:** список задач (звонок / доставка / контакт по платежу) со статусом и
  флагом **«перенесено из-за намаза»** + причина.
- **Бэкенд:** `scheduling`.

### 4.16 Новое напоминание/доставка — `/schedule/new`
- **Элементы:** тип задачи, связанный клиент/договор, желаемое время, длительность
  (для доставки).
- **Логика:** при сохранении система предлагает **ближайший свободный слот мимо
  намаза/джума** и показывает, было ли смещение и почему.
- **Бэкенд:** `scheduling` (алгоритм из skill `namaz-scheduler`).

### 4.17 Реестр садаки — `/charity`
- **Назначение:** прозрачность благотворительных сборов за просрочку (исламская
  корректность и вау-деталь для жюри).
- **Элементы:** таблица (дата, договор, клиент, сумма, статус перечисления);
  итог «собрано / перечислено».
- **Логика:** записи создаются только из карточки договора при начислении садаки
  (**действие владельца**); суммы **не** входят в выручку и **не** влияют на остаток.
  Менеджеру реестр доступен только на просмотр.
- **Бэкенд:** `financing` (реестр садаки).

### 4.18 API-ключи — `/developers/api-keys` (+3)
- **Элементы:** список ключей (название, дата, статус), кнопки «Создать», «Отозвать»,
  «Скопировать» (ключ показывается один раз).
- **Бэкенд:** `iam` / `publicapi`.

### 4.19 Документация API — `/developers/docs` (+3)
- **Элементы:** встроенный **Swagger UI** (или ссылка на `/swagger/`) с эндпоинтами
  публичного API (`/api/v1`, `X-API-Key`); возможность «попробовать» с ключом.
- **Демо:** здесь создаём договор «от имени внешнего маркетплейса».
- **Бэкенд:** `publicapi` (OpenAPI от swaggo).

### 4.20 Вебхуки — `/developers/webhooks` (опционально)
- **Элементы:** список адресов вебхуков, события (платёж принят, договор завершён),
  тест-отправка. **Бэкенд:** `publicapi`.

### 4.21 Настройки → Организация — `/settings/organization`
- Данные организации, валюта по умолчанию. **Бэкенд:** `iam`.

### 4.22 Настройки → Пользователи — `/settings/users`
- Список пользователей, роли (владелец/менеджер), приглашение/отключение.
- **Бэкенд:** `iam`.

### 4.23 Настройки → Намаз — `/settings/prayer-times`
- Координаты (Грозный по умолчанию: 43.3178, 45.6949), **мазхаб** (шафиитский по
  умолчанию), буферы окон до/после молитвы, окно джума (по умолчанию 12:30–14:00).
- **Бэкенд:** `scheduling`.

### 4.24 Настройки → Рассрочка — `/settings/financing`
- Фиксированный **сбор-садака** за просрочку (сумма), периодичность платежей по
  умолчанию, валюта. **Бэкенд:** `financing`.

### 4.25 Настройки → Профиль — `/settings/profile`
- Имя, смена пароля. **Бэкенд:** `iam`.

### 4.26 Системные экраны
- **404 / Маршрут не найден**, общий экран ошибки, состояния «пусто» и «загрузка»
  для всех списков.

## 5. Приоритет для MVP (что собрать к демо в первую очередь)

1. Вход → каркас приложения.
2. Клиенты + Каталог (минимум для договора).
3. **Договоры: мастер создания с предпросмотром графика + карточка договора +
   приём платежа** — это сердце демо.
4. **Календарь/новое напоминание с обходом намаза** — вторая фишка.
5. **Реестр садаки** — короткий, но сильный экран.
6. **Документация API (Swagger)** — для +3.
7. Настройки намаза и рассрочки — по остаточному времени.

Дашборд и часть настроек можно показать в упрощённом виде — баллы дают экраны из
пунктов 3–6.
